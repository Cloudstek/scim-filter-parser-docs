Value Path

The Value Path is like an Attribute Path but is actually a node and can be returned directly by both parsers. It hold both a path and a condition (in brackets).

When using the Path Parser, a Value Path can additionaly contain a sub attribute: name[givenName eq "John" and familyName sw "J"].formatted — in this case the formatted attribute. The Value Path's Attribute Path would then have two parts (names): ['name', 'formatted']. The formatted part will not appear in in Attribute Paths further down the tree as it only applies to the Value Path.
